Phase 01 · Tháng 1
2/10

20% trọng yếu
— Pareto Core

Nắm vững mental model cốt lõi. Sau giai đoạn này, não bạn đã "rewired" để suy nghĩ theo Rust — borrow checker không còn là kẻ thù mà là đồng đội.

Transformation
Từ "tôi viết code và hi vọng nó đúng" → "Compiler là đồng đội. Tôi hiểu tại sao mỗi dòng được chấp nhận hoặc từ chối."
20% tạo ra 80% kết quả
Ownership system CRITICAL
Move semantics, copy vs move, RAII. Đây là linh hồn của Rust — phải internalize, không chỉ memorize.
Borrowing & References CRITICAL
Shared (&T) vs exclusive (&mut T). Chỉ 1 &mut tại một thời điểm. Borrow checker rules trong đầu, không phải trên paper.
Enums & Pattern Matching CRITICAL
Option<T>, Result<T,E>, match exhaustiveness. Đây là error handling của Rust — elegant và type-safe.
Traits cơ bản HIGH
impl Trait for Type. Display, Debug, Clone, Iterator. Trait là ngôn ngữ bên trong ngôn ngữ.
Iterators & Closures HIGH
map, filter, collect, fold. Lazy evaluation. Zero-cost abstraction — không chậm hơn for loop.
Structs & Methods
impl blocks, associated functions (::new), self/&self/&mut self — phân biệt rõ ràng từng loại.
String types
String vs &str. Tại sao có 2 loại? UTF-8, slices. Cực kỳ hay gây confuse lúc đầu — master luôn ở đây.
Cargo workflow
cargo new, build, test, clippy, fmt. Crates.io ecosystem. Đây là superpower của Rust developer.
Lịch học theo tuần
W1
Ownership & Borrowing. Đọc The Book Ch.1–6. Làm Rustlings sections ownership, variables, primitives. Mục tiêu: giải thích được borrow checker cho người khác.
W2
Enums, Pattern Matching, Error Handling. The Book Ch.6, 9. Rustlings enums, error_handling. Viết chương trình đọc file với Result propagation đầy đủ, không dùng unwrap().
W3
Traits & Generics cơ bản. The Book Ch.10, 13. Rustlings traits, generics, iterators. Implement Display, From, Into cho custom types của mình.
W4
Mini project tổng hợp. Build CLI todo-list: đọc/ghi JSON, error handling hoàn chỉnh, iterators, custom structs. Không dùng tutorial — tự design architecture.

Bạn phải làm được tất cả những điều này:
Giải thích ownership, borrowing, lifetimes cho người mới mà không nhìn note
Viết CLI app đọc/ghi file, xử lý error với Result, dùng iterators — không có unwrap()

Phase 02 · Tháng 2–3
3/10

Solid Foundation
— Idiomatic Rust

Rust không còn làm bạn bất ngờ. Bạn viết idiomatic code và bắt đầu có opinions về design. Compiler errors không gây panic nữa.

Transformation
Từ "viết Rust theo kiểu C++/Python" → "Tư duy theo idiom Rust: trait-based design, type-driven correctness, zero-cost là mặc định."
Concepts cần master
Trait objects & dyn CRITICAL
dyn Trait vs impl Trait. Dynamic dispatch khi nào dùng. Object safety rules. Vtable overhead.
Lifetimes nâng cao CRITICAL
Lifetime annotations trong structs và functions. Lifetime elision rules. 'static và tại sao nó không "tốt hơn".
Smart Pointers HIGH
Box<T>, Rc<T>, RefCell<T>, Cell<T>. Interior mutability pattern. Khi nào dùng cái nào và tại sao.
Error handling patterns HIGH
thiserror cho library errors, anyhow cho application errors. Custom error types đẹp và informative.
Generics nâng cao
Multiple trait bounds, where clauses, associated types. Monomorphization và code size trade-offs.
Modules & Visibility
pub, pub(crate), pub(super). Workspace structure. Re-exports. API design fundamentals từ đây.
Testing
Unit tests, integration tests, doc tests. Test organization. Không test = chưa ship được.
macro_rules! cơ bản
Declarative macros. Repetition patterns. Khi nào macro vs function. Macro hygiene basics.
Projects thực hành
Medium
Custom Iterator + Trait hierarchy
Implement một data structure (linked list, trie, hoặc priority queue) với đầy đủ Iterator, IntoIterator, FromIterator. Viết generic API đẹp, test coverage tốt.
Medium
Mini HTTP client từ đầu
Dùng std::net::TcpStream, parse HTTP response thủ công, error handling hoàn chỉnh với thiserror. Không dùng reqwest — học bằng cách build.
Hard
Config parser library — API design exercise
Parse TOML/JSON config file, expose clean public API với proper error types, docstrings, viết doc tests. Publish lên crates.io dù chỉ là 0.1.0. Đây là bài học API design.

Bạn phải làm được tất cả những điều này:

Phase 03 · Tháng 4–6
6/10

Production-Ready
— Ship Real Systems

Bạn xây được systems thật. Async, concurrency, performance bắt đầu click. Đủ để làm việc professional trong một Rust codebase.

Transformation
Từ "biết Rust lý thuyết" → "Ship được production code: async services, CLI tools, xử lý real-world complexity, profile được performance với data."
Concepts cần master
Async / await + Tokio CRITICAL
Future trait internals. async fn, .await, tokio::spawn. Multi-threaded runtime. Biết khi nào KHÔNG dùng async.
Concurrency primitives CRITICAL
Arc<Mutex<T>>, RwLock, channels (mpsc, broadcast, oneshot). Deadlock avoidance. Lock-free basics.
Axum / web ecosystem HIGH
Router, extractors, middleware, state sharing. Tower ecosystem. Serde cho JSON. tracing cho logging.
Performance profiling HIGH
criterion.rs benchmarks. cargo flamegraph. perf. Biết khi nào optimize, khi nào đang premature.
clap — CLI framework
Derive API, subcommands, argument validation. Build CLI tools đạt professional-grade usability.
Database: sqlx
Async, compile-time verified queries. Migrations với sqlx-migrate. Connection pooling. PostgreSQL focus.
Serde ecosystem nâng cao
Custom serializers, flatten, skip_if, remote derives. Multiple format support (JSON, TOML, binary).
Testing nâng cao
Integration tests với real DB, mockall cho mocking, proptest cho property-based testing.
Flagship projects
Hard
REST API server — production-grade
Axum + sqlx + PostgreSQL. JWT authentication, middleware chain, structured error handling, tracing/logging, Docker containerization. Deploy thật sự lên cloud. Không phải tutorial clone — design architecture của bạn.
Hard
CLI tool hữu ích thật sự
Chọn một vấn đề bạn gặp hàng ngày và solve bằng Rust CLI. Publish lên GitHub với README đẹp, release binaries. Người khác phải muốn dùng — đây là chỉ số thật.
Very Hard
Async worker queue
Mini job queue: producers gửi jobs, multiple async workers xử lý concurrent, retry logic với exponential backoff, dead letter queue, Prometheus metrics. Serde + Tokio + channels thuần.

Phase 04 · Tháng 7–18
8/10

Expert Level
— Extend the Language

Unsafe Rust, proc macros, advanced async, systems programming. Bạn bắt đầu design language-level solutions và người khác hỏi ý kiến bạn.

Transformation
Từ "dùng Rust để build apps" → "Extend Rust: viết proc macros người khác dùng, work với unsafe safely, contribute vào ecosystem, mentor người khác một cách tự nhiên."
Deep dive concepts
Unsafe Rust CRITICAL
Raw pointers, unsafe blocks, invariants. Send/Sync manual implementation. Khi nào unsafe justified. FFI với C/C++.
Proc macros CRITICAL
Derive macros, attribute macros, function-like macros. syn + quote crates. TokenStream manipulation từng bước.
Advanced lifetimes HIGH
Variance (covariant, contravariant, invariant). Higher-ranked trait bounds (HRTB). for<'a> syntax và khi nào cần.
Custom async primitives HIGH
Implement simple executor từ đầu. Waker, Context, Poll. Hiểu Tokio internals đủ để debug production issues.
Memory layout & SIMD
repr(C), repr(packed), alignment. Cache-friendly data structures. std::simd basics cho hot paths.
Embedded Rust
no_std environment, embedded-hal, cross-compilation. Blink LED trên real hardware — forces deep understanding.
Compiler internals
Rust compilation pipeline: parsing → HIR → MIR → codegen. cargo-expand để debug macros. Custom clippy lints.
WebAssembly
wasm-bindgen, wasm-pack. Rust → WASM cho browser. Performance characteristics và limitations.
Flagship projects (18 tháng)
Expert
Proc macro library được người khác thật sự dùng
Viết một derive macro non-trivial — Builder pattern với validation, State machine enforcement, hoặc custom serialization. Publish crates.io, support real users, iterate theo feedback. Đây là bước quan trọng nhất của phase này.
Expert
Significant contribution vào crate nổi tiếng
Contribute PR không phải bug fix — thêm feature hoặc cải thiện performance đo được vào tokio, serde, axum, clap, hoặc tương đương. Quy trình review với core maintainers là invaluable learning.
Expert
Systems project: mini DB hoặc mini async runtime
Mini key-value store với WAL và ACID properties, hoặc async executor từ đầu. Unsafe Rust, manual memory management, lock-free data structures. Không cần production-ready — cần deep understanding.

Beyond 18 tháng
10/10

The Summit
— Shape the Language

Không có lộ trình cụ thể nào đưa bạn tới đây. Đây là kết quả của một lifestyle, không phải một course. Bạn không tìm đến summit — summit tìm đến bạn.

Định nghĩa 10/10
Bạn không chỉ viết Rust — bạn shape Rust. Người khác học Rust từ code của bạn, từ crates của bạn, từ bài viết của bạn. Compiler là công cụ, bạn là người hiểu giới hạn và khả năng của công cụ đó sâu hơn 99% người dùng nó.
Dấu hiệu nhận biết 10/10
Published crates được adopt
Crates của bạn có users thật, được referenced trong các project khác, bạn maintain và evolve chúng long-term.
Compiler contributions
Hiểu đủ internals để contribute vào rustc, clippy, hoặc core language tooling. PRs được merge.
API design mastery
Người khác nói code của bạn "feels like Rust." Ergonomic, type-safe, hard to misuse by design.
Teaching & writing
Blog, talks, courses về Rust. Giải thích advanced topics đơn giản và chính xác. Người tìm đến bạn để học.
Unsafe expertise
Biết khi nào unsafe cần thiết, viết sound unsafe code, verify correctness của người khác. Rustonomicon là comfortable reading.
Performance từ first principles
Predict performance từ assembly output. SIMD-aware design. Allocator-conscious APIs. Cache-efficient data structures.
Nguyên tắc transformation không thể đảo ngược
Daily practice
Viết Rust mỗi ngày, dù chỉ 30 phút. Không có ngày nào "off" kéo dài quá 2 ngày. Consistency beats intensity mọi lúc mọi nơi.
Read real code
Mỗi tuần đọc ít nhất 500 lines Rust production code của người khác. Không phải tutorial — codebase thật. tokio, serde, axum.
Ship things
Mỗi tháng phải có ít nhất một "thing" shipped: crate, blog post, PR merged, talk given. Không ship = không grow. Đây là rule cứng.
Teach others
Khi bạn có thể giải thích một concept cho người mới, bạn mới thật sự hiểu nó. Teach early and often, không đợi "đủ giỏi".
Stay curious
Theo dõi Rust RFCs, đọc pre-RFC discussions. Hiểu tại sao language evolves như nó đang làm — không chỉ học syntax mới.
